# Word 文档格式化工具 — 项目架构说明

## 项目用途

输入格式混乱的 `.doc`/`.docx` 文件，自动识别段落层级（H1-H5/Body），通过 Web 界面人工少量核验后，套用 YAML 格式模板，输出整洁规范的文档。

---

## API 端点一览

| 端点 | 方法 | 用途 |
|---|---|---|
| `/` | GET | 返回前端 index.html |
| `/templates` | GET | 列出 templates/ 下所有 .yaml |
| `/upload` | POST | 上传文档，返回段落切片列表（含层级/置信度）|
| `/original-preview` | POST | mammoth 转原文 HTML（Tab2）|
| `/preview` | POST | 单段按模板渲染 HTML（Tab3）|
| `/export` | POST | 按审核后层级生成并下载格式化文档 |

## 处理流程

```
上传文档
  ↓
Step1: 解析 → 逐段执行前置规则 + 四模块判定 → 生成段落切片列表
  ↓
Step2: 前端审核
  ├── Tab1：段落切片表（层级下拉/置信度着色/命中规则）
  ├── Tab2：原文预览（mammoth 转 HTML）
  └── Tab3：格式化预览（实时渲染，拆分段分别渲染标题+Body）
  ↓ 用户点击"确认输出"
Step3: 应用格式模板（内嵌于 Tab3 实时渲染）
  ↓
Step4: 输出格式化文档
  ├── .docx 输入 → 直接输出 .docx
  └── .doc 输入 → win32com 转 .docx 处理后转回 .doc
```
