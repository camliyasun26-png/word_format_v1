# Word 文档格式化工具

输入格式混乱的 Word 文档（`.doc` / `.docx`），自动识别段落层级，经人工少量核验后，套用预设格式模板，输出整洁规范的文档。

---

## 环境要求

- Python 3.11+
- Windows 操作系统（需本机安装 Microsoft Word，用于 `.doc` 格式转换）

---

## 安装

```bash
pip install -r requirements.txt
```

---

## 启动服务

```bash
uvicorn main:app --reload --port 8000
```

启动后在浏览器访问：[http://localhost:8000](http://localhost:8000)

---

## 使用流程

### 1. 上传文档

点击「文件」选择 `.doc` 或 `.docx` 文件，点击「上传解析」。

系统自动完成：
- 段落层级识别（H1–H5 / Body）
- 置信度评估（高/中/低）
- 原文预览加载

### 2. 审核段落层级（Tab1）

| 列 | 说明 |
|---|---|
| # | 段落序号 |
| 原始文本 | 段落内容（悬停显示全文）|
| 层级 | 下拉修改（H1/H2/H3/H4/H5/Body）|
| 置信度 | 🔴 低（需核验）/ 🟡 中 / 🟢 高 |
| 命中规则 | 触发的解析规则说明 |

**低置信度（红色行）** 为重点核验对象，正常文档中占比约 5% 以下。

修改层级下拉后，Tab3 格式化预览实时更新。

### 3. 原文预览（Tab2）

展示原始文档的排版效果，只读。

### 4. 格式化预览（Tab3）

展示套用格式模板后的效果，随 Tab1 层级修改实时更新，所见即所得。

### 5. 切换格式模板

页面顶部「模板」下拉可切换 `templates/` 目录下的所有 `.yaml` 模板，切换后 Tab3 立即用新模板重新渲染。

### 6. 导出文档

确认预览效果后点击「确认输出」，自动下载格式化后的文档。

- `.docx` 输入 → 输出 `原文件名_formatted.docx`
- `.doc` 输入 → 输出 `原文件名_formatted.doc`

## 格式模板

模板文件存放于 `templates/` 目录，格式为 YAML。

**内置模板：`report_cn.yaml`**（中文公文格式）

| 层级 | 字号 | 加粗 | 对齐 | 首行缩进 |
|---|---|---|---|---|
| H1 | 18pt | ✓ | 居中 | 0 |
| H2 | 14pt | ✓ | 两端对齐 | 24pt（约2字符）|
| H3 | 12pt | ✓ | 两端对齐 | 24pt（约2字符）|
| H4 | 12pt | ✗ | 两端对齐 | 48pt（约4字符）|
| Body | 12pt | ✗ | 两端对齐 | 24pt（约2字符）|

页边距：上下 2.54cm，左右 3.17cm，全部层级使用仿宋_GB2312，行距 1.5。

### 添加自定义模板

在 `templates/` 目录下新建 `.yaml` 文件，参照 `report_cn.yaml` 结构填写各层级配置，重启服务后即可在模板下拉中选择。

## 层级判定规则说明

系统按以下顺序识别每个段落的层级（加粗的 run 触发判定）：

| 模式 | 层级 |
|---|---|
| `第N章` / `第一章` + 加粗 | H1 |
| `一、二、` 等中文序号 + 加粗 | H2 |
| `（一）（二）` 等括号汉字序号 + 加粗 | H3 |
| `1.` `2.` 单级数字 + 加粗 | H3 或 H4（向上跳过同型回溯前文判定） |
| `1.1` `2.3` 两级数字 + 加粗 | H4 |
| `1.1.1` 三级数字 + 加粗 | H5 |
| `(1)` `(a)` 括号编号 + 加粗 | 上一层非同型标题下移一层（默认 H4） |
| `1)` `a)` 单括号编号 + 加粗 | 上一层非同型标题下移一层（默认 H5） |
| 数字开头 + 冒号 + 加粗（冒号后 > 34 字）| 标题+正文自动拆分 |
| 无任何加粗 | Body |

**容错说明：**
- 编号内外的全角/半角空白均不影响匹配（如 `第 一 章`、`（ 1 ）`、`1． `）。
- 段首若存在 Word 编辑残留的空白 run（空文本且 bold 属性可能为 None），解析器会跳过这些 run 寻找真实的首个加粗 run，避免被误判为 Body。
- 无 H1 的文档同样支持模块二的"上下文依赖"判定，首个出现的 H2-H5 也会开启章节上下文。
